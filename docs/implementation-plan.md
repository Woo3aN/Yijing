# 分步执行计划

## 阶段划分原则

- 每阶段独立可验证
- 先核心后外围
- 数据层优先（因为上层都依赖它）
- 每完成一个阶段，软件都能跑起来看到进展

---

## 阶段 0：环境准备

**目标**：开发环境就绪，能运行一个空白窗口

### 步骤
1. 确认 Python 3.11+ 已安装
2. 创建虚拟环境 `python -m venv venv`
3. 激活虚拟环境
4. 安装依赖：`pip install ttkbootstrap openai`
5. 验证 tkinter 可用：`python -c "import tkinter; tkinter.Tk()"`

### 验证标准
- 弹出一个空白 tkinter 窗口即成功

### 产出文件
- `requirements.txt`

---

## 阶段 1：数据基础 — hexagrams.json

**目标**：完成 64 卦完整原文数据

### 步骤
1. 先完成前 2 卦（乾、坤）验证数据结构
2. 完成全部 64 卦的数据录入
3. 数据校对（卦名、卦辞、爻辞是否匹配）

### 验证标准
- JSON 格式合法，能被 Python `json.load()` 正确解析
- 64 卦全部存在，每卦 6 个爻
- binary 字段与 lines 的阴阳一致

### 产出文件
- `src/data/hexagrams.json`

---

## 阶段 2：核心引擎 — 数据加载 + 起卦算法

**目标**：能用代码加载卦象数据，执行起卦并得到正确卦名

### 步骤
1. 实现 `text_loader.py`：加载 JSON、资源路径解析（开发模式 & 打包模式）
2. 实现 `hexagram.py`：数据模型、binary→卦序查表
3. 实现 `divination.py`：
   - `coin_method()` 三铜钱法
   - `random_method()` 随机数法
   - `lines_to_hexagram()` 判定本卦和变卦
4. 写一个简单的命令行测试脚本验证

### 验证标准
- 运行 100 次三铜钱法，确认 6/7/8/9 分布合理
- 手动输入 64 种 binary 值，确认每种都查到正确卦名
- 有变爻时正确识别变卦

### 产出文件
- `src/core/text_loader.py`
- `src/core/hexagram.py`
- `src/core/divination.py`

---
