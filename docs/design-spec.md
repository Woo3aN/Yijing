# UI 设计规范

## 设计原则
- **简洁优先**：不堆砌信息，每个页面一个核心任务
- **中文界面**：所有文本使用中文
- **仪式感**：起卦过程有动画，结果展示有层次

## 全局设置

| 项目 | 值 |
|------|-----|
| 主题 | ttkbootstrap "litera"（浅色，干净） |
| 默认字体 | Microsoft YaHei 10pt |
| 标题字体 | Microsoft YaHei 14pt bold |
| 卦符字体 | Microsoft YaHei 36pt |
| 窗口初始大小 | 900 × 700 px |
| 窗口最小大小 | 600 × 500 px |
| 窗口标题 | 易经占卜 |

### 交互流程

1. 用户输入问题 → 选择起卦方法
2. 播放松爻动画 → 显示完整六爻
3. 自动显示卦象结果（含本卦、变卦、变爻详情）
4. 用户可选点击"AI 解读"
5. 结果自动保存到历史记录

## 颜色与图标

- 主要按钮：Bootstrap primary 蓝色
- 次要按钮：Bootstrap secondary 灰色
- 成功/确认：Bootstrap success 绿色
- 危险/删除：Bootstrap danger 红色
- 变爻标记：橙色或红色圆形 ●
- 不变爻：默认文本色
- 老阳 (9)：橙色 ○
- 老阴 (6)：橙色 ×

## 状态提示规范

| 场景 | 提示文本 | 样式 |
|------|----------|------|
| API Key 未配置 | "请先在「设置」中配置 API 密钥" | 灰色提示文本 |
| API 调用中 | "正在请求 AI 解读..." | 加载动画 |
| API 调用成功 | 显示返回内容 | 正常文本 |
| API 调用失败 | "AI 请求失败：{具体错误}" | 红色警告 |
| 测试连接成功 | "✓ 连接成功！" | 绿色文本 |
| 测试连接失败 | "✗ 连接失败：{错误信息}" | 红色文本 |
| 保存成功 | "设置已保存" | 绿色简短提示 |
| 历史记录已删除 | "记录已删除" | 提示信息 |
